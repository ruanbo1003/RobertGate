# UI 风格规范

## 核心原则
创造有创意、独特的前端，让人惊喜和愉悦。
避免"AI 流水线"美学——不要收敛到通用的、可预测的输出。
每次设计都应做出意想不到的选择，真正符合当前项目的情境。

## 字体
- 选择美观、独特、有趣的字体
- 严禁使用：Arial、Inter、Roboto、系统字体（San Francisco、Segoe UI 等）
- 严禁重复使用同一字体家族（如 Space Grotesk）跨功能收敛
- 从 Google Fonts 或 Bunny Fonts 中选择能提升美学的独特字体
- 标题与正文使用不同字体，制造对比和层次感

## 颜色与主题
- 使用 CSS 变量维护一致的色彩系统
- 主导色 + 鲜明强调色，避免平淡均匀分布的调色板
- 严禁：白底紫色渐变等陈词滥调配色
- 从 IDE 主题、文化美学、游戏视觉等非常规来源汲取灵感
- 在浅色/深色主题之间保持变化，不要每次都默认深色

## 动效
- 为关键时刻添加动画，而非随意散布微交互
- 优先：精心编排的页面加载（带交错显示 staggered reveal）
- HTML/CSS 优先使用纯 CSS 方案（@keyframes、transition）
- React 项目使用 Motion 库（framer-motion）
- 高影响力时刻：页面加载、模态框出现、列表进入

## 背景
- 创造氛围和深度，禁止使用纯色背景作为默认选项
- 可选方案：CSS 渐变叠加、几何图案、噪点纹理、情境效果
- 背景需与整体美学一致，不能突兀

## 布局
- 避免可预测的布局和组件模式
- 尝试不对称布局、打破网格、大胆的留白
- 所有页面根容器：width: 100%; max-width: 100%; overflow-x: hidden
- 统一使用 box-sizing: border-box
- 移动端优先

## 禁止清单
- Inter、Roboto、Arial、系统字体
- 白底紫色渐变
- 千篇一律的卡片 + 圆角 + 阴影组合
- 没有情境特色的通用组件样式
- 跨功能使用相同字体、相同配色
